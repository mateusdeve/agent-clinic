# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 1 scope: public landing page only. All contracts below apply exclusively to the `(landing)` route group. No authenticated UI is built in this phase.

Source of truth: `frontend/site.html`. All tokens and copy are extracted directly from that file.

## Interaction Contract

### CTA Buttons

Two CTA button styles:

**Primary button (`.btn-primary` equivalent):**
- Background: green-500 (`#2e9e60`)
- Text: white, DM Sans 15px weight 600
- Padding: 14px 28px
- Border radius: 12px
- Hover: bg green-600, translateY(-2px), box-shadow `0 8px 24px rgba(46,158,96,0.3)`
- Transition: background 0.2s, transform 0.15s, box-shadow 0.2s

**Nav CTA button:**
- Background: green-500
- Text: white, DM Sans 14px weight 500
- Padding: 10px 24px
- Border radius: 100px (pill)
- Hover: bg green-600, translateY(-1px)
- Transition: background 0.2s, transform 0.15s

**Large button (`.btn-lg` equivalent — FinalCTA section):**
- Same as primary button but padding: 16px 32px, font-size 16px, border-radius 14px

Both primary CTAs are `<a>` anchor tags pointing to `https://wa.me/${NEXT_PUBLIC_WHATSAPP_NUMBER}?text=${encodedMessage}`. No form submission. `target="_blank"` + `rel="noopener noreferrer"`.

**Note:** site.html uses an email waitlist form. Per STATE.md decision "Landing page redirects to WhatsApp — no email funnel", the input fields are removed. The CTA buttons link directly to WhatsApp.

### Navigation

- Fixed position, z-index 100, top 0
- Background: `rgba(255,255,255,0.88)` with `backdrop-filter: blur(12px)`
- Border-bottom: 1px solid gray-200
- Logo: DM Serif Display 24px, color green-600, italic span color green-400
- Nav CTA: pill button linking to WhatsApp

### CSS Animations

Three animation types from site.html (implemented as CSS keyframes in globals.css):

| Name | Effect | Applied to |
|------|--------|-----------|
| `fadeUp` | opacity 0→1, translateY 24px→0, duration 0.6s | Hero content elements (staggered 0s, 0.1s, 0.2s, 0.3s, 0.4s, 0.5s delays) |
| `fadeMsg` | opacity 0→1, translateY 8px→0, duration 0.4s | Chat message bubbles (staggered 0.5s, 0.9s, 1.3s, 1.7s delays) |
| `bounce` | translateY 0→-6px→0, duration 1.2s infinite | Typing indicator dots |
| `pulse` | opacity 1→0.3→1, duration 2s infinite | Badge status dot |

Scroll-triggered entrance for `.feat-card` and `.step` elements: IntersectionObserver sets `opacity: 0; transform: translateY(24px)` initially, transitions to `opacity: 1; transform: none` on intersect. Since this requires `useEffect`, the FeaturesSection and HowItWorksSection cards require `"use client"` OR can be implemented with CSS `@keyframes` triggered by `animation-play-state` via Intersection Observer. Recommend CSS-only approach using `@starting-style` or a lightweight client wrapper component to avoid marking entire sections as client.

### Hover States

| Element | Hover behavior |
|---------|----------------|
| Feature cards | translateY(-4px), box-shadow `0 16px 40px rgba(46,158,96,0.1)` |
| CTA buttons | translateY(-2px), bg darken to green-600, shadow |
| Nav CTA | translateY(-1px), bg darken to green-600 |

### Responsive Breakpoint

Single breakpoint at 900px (`max-width: 900px` → maps to Tailwind `lg:` prefix at 1024px, or use custom `@media (max-width: 900px)` via Tailwind's arbitrary variant).

| Change at 900px | Desktop | Mobile |
|-----------------|---------|--------|
| Hero layout | 2-column grid | 1-column |
| Hero visual (chat card) | Visible | Hidden (`hidden lg:block`) |
| Stats grid | 3 columns | 1 column |
| Features grid | 3 columns | 1 column |
| Steps grid | 4 columns | 2 columns |
| Steps connector line | Visible | Hidden |
| All section padding | 48px horizontal | 24px horizontal |

`overflow-x: hidden` on both `html` and `body`. Mandatory to prevent radial gradient blobs from causing horizontal scroll.
