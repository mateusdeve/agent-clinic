       ```python
       import os, logging
       from datetime import datetime, timedelta, timezone
       from typing import Annotated, Optional
       from contextlib import contextmanager

       import jwt
       import psycopg2
       from fastapi import APIRouter, Depends, HTTPException, status
       from fastapi.security import OAuth2PasswordBearer
       from jwt.exceptions import InvalidTokenError
       from pwdlib import PasswordHash
       from pydantic import BaseModel
       from dotenv import load_dotenv

       load_dotenv()

       logger = logging.getLogger("agent-clinic.auth")

       SECRET_KEY = os.getenv("JWT_SECRET_KEY")
       ALGORITHM = "HS256"
       ACCESS_TOKEN_EXPIRE_MINUTES = 30
       REFRESH_TOKEN_EXPIRE_DAYS = 7

       password_hash = PasswordHash.recommended()
       router = APIRouter(prefix="/auth", tags=["auth"])
       ```

       **Models:**
       - `LoginRequest(BaseModel)`: email: str, password: str
       - `TokenResponse(BaseModel)`: access_token, refresh_token, token_type, user_id, email, name, role, tenant_id (all str)
       - `UserResponse(BaseModel)`: id, email, name, role, tenant_id (all str)

       **Helper — _get_db():** Reuse the exact same pattern from `src/tools/patients.py` (raw psycopg2 contextmanager with `DATABASE_URL`).

       **Helper — create_token(data: dict, expires_delta: timedelta) -> str:**
       Uses `jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)`.

       **Helper — hash_password(password: str) -> str:**
       Uses `password_hash.hash(password)`.

       **Helper — verify_password(plain: str, hashed: str) -> bool:**
       Uses `password_hash.verify(plain, hashed)`.

       **POST /auth/login:**
       1. Fetch user from users table by email: `SELECT id, email, name, hashed_password, role, tenant_id, is_active FROM users WHERE email = %s`
       2. If no user found or is_active is False, raise 401 "Credenciais invalidas"
       3. Verify password with `verify_password(request.password, user_row.hashed_password)`
       4. If password wrong, raise 401 "Credenciais invalidas"
       5. Create access_token with claims: `{"sub": str(user_id), "tenant_id": str(tenant_id), "role": role}`
       6. Create refresh_token with claims: `{"sub": str(user_id), "type": "refresh"}`
       7. Return TokenResponse

       **POST /auth/refresh:**
       1. Accept `RefreshRequest(BaseModel)` with `refresh_token: str`
       2. Decode refresh token, verify `type == "refresh"`
       3. Fetch user from DB by `sub` (user_id) to get current role and tenant_id
       4. Issue new access_token (NOT new refresh_token — rotation in v2)
       5. Return `{"access_token": ..., "token_type": "bearer"}`

       **GET /auth/me:**
       1. Depend on `get_current_user` (from deps.py, created in Task 2)
       2. Return UserResponse with id, email, name, role, tenant_id

       **POST /auth/seed (temporary dev endpoint):**
       Creates a seed admin user for testing. Accepts `{email, password, name}`.
       Hashes password, inserts into users with role='admin', tenant_id='00000000-0000-0000-0000-000000000001'.
       Guard with `if os.getenv("ENVIRONMENT") != "production"` to prevent running in prod.

       ```python
       import os, logging
       from typing import Annotated
       from contextlib import contextmanager

       import jwt
       import psycopg2
       from fastapi import Depends, HTTPException, status
       from fastapi.security import OAuth2PasswordBearer
       from dotenv import load_dotenv

       load_dotenv()

       logger = logging.getLogger("agent-clinic.deps")
       ```

       **get_current_user(token: Annotated[str, Depends(oauth2_scheme)]) -> dict:**
       - Decode JWT with PyJWT
       - Extract `sub` (user_id), `tenant_id`, `role` from payload
       - If any missing or decode fails, raise 401 "Credenciais invalidas"
       - Return `{"user_id": sub, "tenant_id": tenant_id, "role": role}`

       **get_current_tenant(user: dict = Depends(get_current_user)) -> str:**
       - Per D-13/TENANT-03: Extract tenant_id from JWT, never from request params
       - Return `user["tenant_id"]`

       **require_role(*roles: str):**
       - Dependency factory that returns an async function
       - The inner function depends on `get_current_user`
       - Checks `user["role"] in roles`
       - If not, raise 403 "Permissao insuficiente"
       - Returns user dict if authorized
       - Per D-07, this is the source-of-truth role enforcement

       **get_db_for_tenant(tenant_id: str = Depends(get_current_tenant)):**
       - Context manager (use as generator dependency)
       - Creates psycopg2 connection from DATABASE_URL
       - Executes `SET LOCAL app.tenant_id = %s` (per research Pattern 6, use SET LOCAL not SET)
       - Yields connection
       - Closes in finally block
       - Per API-03, this is the automatic multi-tenancy middleware
