- ScheduleUpdate(BaseModel): slots (list[ScheduleSlot])

Endpoints:

1. **GET /** — List doctors (DOC-01)
   - Dependencies: require_role("admin", "recepcionista", "medico")
   - Query params: search (Optional[str]), page (int=1), per_page (int=20)
   - SQL: SELECT id, nome, especialidade, crm, user_id, is_active FROM doctors WHERE 1=1 [+search on nome/especialidade] ORDER BY nome LIMIT/OFFSET
   - Note: doctors table may not have is_active column. Read src/tools/doctors.py to check. If missing, omit from SELECT.
   - Return PaginatedResponse

2. **POST /** — Create doctor (DOC-02)
   - Dependencies: require_role("admin")
   - Body: DoctorCreate
   - SQL: INSERT INTO doctors (nome, especialidade, crm, user_id, tenant_id) VALUES (%s, %s, %s, %s, current_setting('app.tenant_id')::uuid) RETURNING *
   - conn.commit()
   - Return 201

3. **PUT /{doctor_id}** — Edit doctor (DOC-02)
   - Dependencies: require_role("admin")
   - Body: DoctorUpdate
   - Dynamic SET clause from non-None fields
   - 404 if not found
   - conn.commit()

4. **GET /{doctor_id}/schedules** — Get availability (DOC-03 read)
   - Dependencies: require_role("admin", "recepcionista", "medico")
   - SQL: SELECT id, doctor_id, day_of_week, start_time::text, end_time::text FROM doctor_schedules WHERE doctor_id = %s ORDER BY day_of_week, start_time
   - Return list of schedule slots

5. **PUT /{doctor_id}/schedules** — Set availability grid (DOC-03 write)
   - Dependencies: require_role("admin")
   - Body: ScheduleUpdate (list of ScheduleSlot)
   - Strategy: DELETE all existing schedules for this doctor, then INSERT new ones in batch
   - SQL: DELETE FROM doctor_schedules WHERE doctor_id = %s; then for each slot: INSERT INTO doctor_schedules (doctor_id, day_of_week, start_time, end_time, tenant_id) VALUES (%s, %s, %s, %s, current_setting('app.tenant_id')::uuid)
   - conn.commit()
   - Return updated list

6. **GET /{doctor_id}/blocked-slots** — Get blocked slots (AGENDA-06 read)
   - Dependencies: require_role("admin", "recepcionista")
   - SQL: SELECT * FROM blocked_slots WHERE doctor_id = %s ORDER BY blocked_date, start_time
   - Return list

7. **POST /{doctor_id}/blocked-slots** — Block time slot (AGENDA-06 write)
   - Dependencies: require_role("admin")
   - Body: { blocked_date: str, start_time: Optional[str], end_time: Optional[str], reason: Optional[str] }
   - SQL: INSERT INTO blocked_slots (doctor_id, blocked_date, start_time, end_time, reason, tenant_id) VALUES (...)
   - conn.commit()
   - Return 201

8. **DELETE /blocked-slots/{slot_id}** — Unblock time slot (AGENDA-06)
   - Dependencies: require_role("admin")
   - SQL: DELETE FROM blocked_slots WHERE id = %s RETURNING *
   - 404 if not found
   - conn.commit()

Endpoints:

1. **GET /** — List users (USER-01)
   - Dependencies: require_role("admin")
   - Query params: page (int=1), per_page (int=20)
   - SQL: SELECT id, email, name, role, is_active, created_at FROM users ORDER BY name LIMIT/OFFSET
   - Return PaginatedResponse

2. **POST /** — Create user (USER-02)
   - Dependencies: require_role("admin")
   - Body: UserCreate
   - Validate role is one of: admin, recepcionista, medico
   - Hash password: `hasher.hash(body.password)`
   - SQL: INSERT INTO users (email, name, hashed_password, role, tenant_id, is_active) VALUES (%s, %s, %s, %s, current_setting('app.tenant_id')::uuid, true) RETURNING id, email, name, role, is_active, created_at
   - Handle unique constraint violation on email: catch psycopg2 IntegrityError, return 409 "Email ja cadastrado"
   - conn.commit()
   - Return 201

3. **PATCH /{user_id}/role** — Change role (USER-03)
   - Dependencies: require_role("admin")
   - Body: UserRoleUpdate
   - Validate role value
   - Prevent admin from changing their own role (compare user_id param with current_user["user_id"])
   - SQL: UPDATE users SET role = %s WHERE id = %s RETURNING *
   - 404 if not found
   - conn.commit()

4. **PATCH /{user_id}/status** — Activate/deactivate (USER-04)
   - Dependencies: require_role("admin")
   - Body: UserStatusUpdate
   - Prevent admin from deactivating themselves
   - SQL: UPDATE users SET is_active = %s WHERE id = %s RETURNING *
   - 404 if not found
   - conn.commit()

5. **POST /{user_id}/reset-password** — Admin-managed password reset (USER-05)
   - Dependencies: require_role("admin")
   - Body: PasswordReset
   - Hash: `hasher.hash(body.new_password)`
   - SQL: UPDATE users SET hashed_password = %s WHERE id = %s RETURNING id
   - 404 if not found
   - conn.commit()
