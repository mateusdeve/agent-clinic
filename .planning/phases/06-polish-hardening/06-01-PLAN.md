**DataTable responsive wrapper:**
In `frontend/src/components/dashboard/DataTable.tsx`, wrap the `<Table>` element (and its surrounding border div) in a responsive scroll container. Find the outermost div that directly wraps `<Table>` and add `overflow-x-auto` to it. If the table is inside a `<div className="rounded-md border">`, change it to `<div className="rounded-md border overflow-x-auto">`. This ensures tables scroll horizontally at 375px instead of causing page overflow. Do NOT add overflow-x-auto to any parent that contains other content besides the table.

**InboxPanel mobile layout:**
In `frontend/src/app/(dashboard)/whatsapp/_components/InboxPanel.tsx`, the current 3-column layout (ConversationList | ConversationThread | PatientSidebar) must collapse on mobile:

1. Add mobile state awareness: the 3 columns should be visible side-by-side at `md:` breakpoint and above. Below md, show only one panel at a time.
2. Wrap the 3-column flex container. The existing classes likely have `flex` for the columns. Add responsive behavior:
   - ConversationList column: add `md:block` and conditionally show/hide. On mobile (`< md`), show by default. When a conversation is selected on mobile, hide the list.
   - ConversationThread column: add `md:block` and conditionally show/hide. On mobile, show only when a conversation is selected.
   - PatientSidebar: keep `hidden md:block` (hide completely on mobile; sidebar info is secondary).
3. Add a back button in ConversationThread header (visible only on mobile: `md:hidden`): `<button className="md:hidden p-1 mr-2" onClick={clearSelectedConversation}><ArrowLeft className="size-5" /></button>`. Import `ArrowLeft` from `lucide-react`.
4. The mobile flow is: list (default) -> tap conversation -> thread (with back button) -> tap back -> list.
5. Use CSS classes for the responsive show/hide, not JS media queries. Pattern: on the list wrapper, add class logic like `${selectedConversation && !isDesktop ? 'hidden' : 'flex'} md:flex` where the mobile hide is CSS-driven. However, since we can't easily detect `md:` in JS, use a simpler approach: add `hidden md:flex` to the list wrapper when a conversation is selected (controlled by existing state), and `flex md:flex` when no conversation selected. For the thread wrapper, do the inverse: `flex md:flex` when conversation selected, `hidden md:flex` when not.
